對 Windows / MIS 環境的導入判斷

1. 先定義執行邊界 Native Windows 與 WSL2 不要混用路徑、Git、Node、Python 或 package manager。Codex 在哪裡跑，測試與依賴就應該在同一個環境內。
2. 沙盒與權限要先驗證 Windows 原生沙盒是安全重點。企業 GPO、防火牆、代理伺服器或 endpoint protection 可能影響 setup、網路與升權流程。
3. 手機遠端不是檔案上手機 搜尋結果提到手機可引導或檢視 Codex 任務，但本機檔案、憑證與權限仍應留在 Codex 執行的 Windows 裝置或核准環境。
